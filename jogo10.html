Organização das folhas de eventos e fases do jogo

👉 A folha de eventos principal é "Eventos Gerais", que gerencia as mecânicas centrais do jogo. As outras folhas de eventos são específicas para cada fase, como "Fase 1", "Fase 2" e "Fase 3".

👉 Cada fase tem seus próprios eventos e lógica, permitindo uma organização clara e modular do código do jogo.

👉 LEMBRE DE SEMPRE VERIFICAR QUAL FOLHA DE EVENTOS ESTÁ SELECIONADA EM CADA LAYOUT (Fase)

!!CRIE AS VARIÁVEIS 😡!!

Eventos gerais

Eventos de Animação

👉 Verificação de morte: Se a animação "Death" não estiver em execução, o sistema continua avaliando os outros estados.

👉 Movimento: Quando o player está se movendo, a animação muda para "Walk" (andar).

👉 Parado: Caso contrário, se não estiver andando, a animação é definida como "Idle" (parado).

👉 No ar: Se o player não estiver no chão, a animação muda para "Jump" (pulo).

👉 Colisão com fogo: Ao encostar no fogo, o player executa a animação "Death", ignora comandos da plataforma, aguarda 1 segundo e o layout é reiniciado.

Eventos de Som e Música

👉 Pulo: Ao pressionar ↑ e estando no chão, toca o som "Jump 1".

👉 Pouso: Quando o player aterrissa, toca o som "Hit damage 1".

👉 Início do layout: Se musica for falso, define como verdadeiro e toca a música "Forgotten Station" em loop.

Eventos de Definições

👉 Escalas: Player definido com escala 1.6 e Porta com escala 2.

👉 Player: Força do pulo ajustada para 360 e velocidade máxima da plataforma para 260.

👉 Inimigos: Campo de visão ajustado para 50.

👉 Movimentação dos inimigos: Velocidade máxima configurada em 2000 e desaceleração em 6000.

👉 Campo de visão: Adicionado o obstáculo CarinhaAzul.

👉 Organização: Player movido para o topo da camada.

👉 Ao destruir Player: Reinicia o layout.

Eventos de Transição e Espelhamento

👉 Porta: Quando o Player colide com a Porta (na animação "Aberto"), a variável Fase é aumentada em +1.

👉 JavaScript: Um código é executado para controlar a troca de fases. Ele obtém o valor da variável global Fase, monta dinamicamente o nome do próximo layout e, em seguida, troca para esse layout:


const faseAtual = runtime.globalVars.Fase;
const proximoLayout = "Fase " + faseAtual;
runtime.goToLayout(proximoLayout);

👉 Espelhamento: Se a tecla ← é pressionada, o Player é definido como espelhado. Se a tecla → é pressionada, ele volta para não espelhado.

Fase 1

Eventos da Plataforma que Desaparece

👉 Evento 1: Quando o layout inicia, a plataformaSome é configurada com Sólido desabilitado, impedindo que o jogador interaja com ela antes do funcionamento previsto.

👉 Evento 2: Se a plataformaSome estiver dentro do Campo de Visão do personagem CarinhaAzul, o sistema detecta e ativa a lógica de desaparecimento.

👉 Evento 3: A cada tick, enquanto a opacidade da plataformaSome for diferente de zero, o sistema reduz gradualmente a opacidade em pequenos intervalos, aguardando 0.1 segundos entre cada redução, até que a plataforma desapareça completamente.
