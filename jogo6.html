📘 Documentação do Projeto - Player vs Player

Tutorial

  • Um player ataca e o outro tenta defender com a tecla Q (p1) ou P(p2).
  • Assim que o ataque (espaço) é realizado a vez do player muda.
  • Preste atenção no outro jogador (do seu lado) para tentar prever quando o ataque vai acontecer.
  • Quem tirar toda vida do outro player primeiro ganha.

Código do jogo

1 - Sistema: Ao iniciar layout          | Player1: Definir escala para 1.3  
                                        | Player2: Definir escala para 1.3  
                                        | Player2: Definir Espelhado  
                                        | Sistema: Definir VezDoJogador para escolher(1,2)  
                                        | Audio: Tocar 2 - Heavy Combat - Elven Battle (loop) com repetição, volume 10 dB (pan estéreo 0, tag "musica")

👉 Quando o layout inicia, o tamanho (escala) do Player1 e do Player2 é definido para 1.3.

👉 O Player2 é configurado como espelhado, para ficar virado para o lado oposto do Player1.

👉 O sistema escolhe aleatoriamente quem começa o jogo, podendo ser o Jogador 1 ou o Jogador 2.

👉 A música Heavy Combat - Elven Battle é reproduzida em loop infinito.

2 - Teclado: Ao pressionar Barra de espaço   | Sistema: Aguardar 0.12 segundos (usar escala de tempo: Verdadeiro)  
    Sistema: Definir ataque                  |
    Sistema: VidaP1 ≠ 0                      |
    Sistema: VidaP2 ≠ 0                      |

	3 - Sistema: VezDoJogador = 1         | Sistema: Chamar jogadaP1  
                                              | Sistema: Definir ataque para Verdadeiro

	4 - Sistema: VezDoJogador = 2         | Sistema: Chamar jogadaP2  
                                              | Sistema: Definir ataque para Verdadeiro

👉 Quando o jogador pressiona a Barra de Espaço, o sistema aguarda 0.12 segundos antes de executar as próximas ações.

👉 Após a espera, o sistema ativa a variável ataque.

👉 Essa ação só acontece se a Vida do Jogador 1 e a Vida do Jogador 2 forem diferentes de 0 (ou seja, ambos ainda estão vivos).

👉 Se for a vez do Jogador 1, o sistema chama a função jogadaP1 e define o ataque como Verdadeiro.

👉 Se for a vez do Jogador 2, o sistema chama a função jogadaP2 e também define o ataque como Verdadeiro.

 
5 - Teclado: Ao pressionar Q                    | Player1: Definir animação para "block" (executar do início)  
    Player1: A animação "idle" está em execução | Sistema: Definir p1Block para Verdadeiro  
    Sistema: VezDoJogador = 2                   | Sistema: Aguardar 1 segundo (usar escala de tempo)  
    Sistema: [X] Definição p1Block              | Sistema: Definir p1Block para Falso  
    Sistema: VidaP1 ≠ 0                         

6 - Teclado: Ao pressionar P                    | Player2: Definir animação para "block" (executar do início)  
    Player2: A animação "idle" está em execução | Sistema: Definir p2Block para Verdadeiro  
    Sistema: VezDoJogador = 1                   | Sistema: Aguardar 1 segundo (usar escala de tempo)  
    Sistema: [X] Definição p2Block              | Sistema: Definir p2Block para Falso  
    Sistema: VidaP2 ≠ 0

👉 Quando o jogador pressiona Q (Player1) ou P (Player2), o personagem executa a animação "block", mas apenas se estiver na animação "idle" (parado).

👉 O sistema define a variável de bloqueio (p1Block ou p2Block) como Verdadeiro.

👉 Se for a vez do outro jogador, o sistema aguarda 1 segundo e em seguida desativa o bloqueio, definindo a variável como Falso.

👉 Esse bloqueio só funciona enquanto a vida do jogador for diferente de 0.

🛡️ Dica: O bloqueio dura apenas 1 segundo e só pode ser ativado quando o personagem está parado.

7 - Player1: Quando animação "ataque" terminar       | Player1: Definir animação para "Idle" (executar do início)  
    Player1: **OR** Quando animação "block" terminar |  
    Player1: **OR** Quando animação "hit" terminar   |  

8 - Player2: Quando animação "ataque" terminar       | Player2: Definir animação para "Idle" (executar do início)  
    Player2: **OR** Quando animação "block" terminar |  
    Player2: **OR** Quando animação "hit" terminar   |

👉 Quando a animação de "ataque", "block" ou "hit" termina (tanto para o Player1 quanto para o Player2), o personagem volta automaticamente para a animação "idle", iniciando-a do começo.

🎬 Isso garante que, após qualquer ação, o jogador sempre retorne ao estado parado e pronto para a próxima jogada.

    	     Player1: [X] Animação "block" está em execução   | Sistema: Subtrair 1 de VidaP1
                                                              | Áudio: Tocar "26_sword_hit_1" (volume 0 dB, sem repetição)

👉 Quando é a vez de um jogador (VezDoJogador = 1 ou 2), a função jogada é chamada.

👉 O jogador ativo toca o som "11_human_damage_1", é movido para o topo da camada e inicia a animação "ataque".

👉 O sistema aguarda 1 segundo, depois passa a vez para o outro jogador e define o ataque como Falso.

👉 Durante a animação de ataque, se o oponente não estiver em bloqueio, o sistema:

  • 🎞️ Define a animação do oponente como "hit".
  • ❤️ Subtrai 1 ponto de vida do oponente.
  • 🔊 Toca o som "26_sword_hit_1" (sem repetição).

👉 Caso o oponente esteja em block, o ataque não causa dano.

⚔️ Assim, cada jogador alterna sua vez de atacar, e o dano só é aplicado se o adversário não estiver bloqueando no momento.

15 - Sistema: A cada tick                    | BarraDeVida: Definir progresso para VidaP1
                                             | BarraDeVida: Definir progresso para VidaP2
                                             | Áudio: Definir volume de "musica" para -15 dB  

16 - Sistema: VezDoJogador = 1               | botaoQ: Definir opacidade para 0
     Sistema: Ativa uma vez                  | botaoP: Definir opacidade para 100
                                             | texto: Definir texto para "Player 1"

17 - Sistema: VezDoJogador = 2               | botaoQ: Definir opacidade para 100
     Sistema: Ativa uma vez                  | botaoP: Definir opacidade para 0
                                             | texto: Definir texto para "Player 2"

👉 A cada tick do jogo, as barras de vida são atualizadas para refletir os valores de VidaP1 e VidaP2. Além disso, o volume da música de fundo é ajustado para -15 dB.

👉 Quando é a vez do Jogador 1:

  • O botão Q fica invisível (opacidade 0).
  • O botão P fica visível (opacidade 100).
  • O texto exibido é "Player 1".

👉 Quando é a vez do Jogador 2:

  • O botão P fica invisível (opacidade 0).
  • O botão Q fica visível (opacidade 100).
  • O texto exibido é "Player 2".

🎮 Dessa forma, os botões e o texto na interface indicam claramente de quem é a vez, enquanto a barra de vida e a música são sempre mantidas atualizadas.


18 - Sistema: VidaP1 = 0                     | Player1: Definir animação para "morte" (executar do início)
     Sistema: Ativa uma vez                  | Sistema: Aguardar 2 segundos (escala de tempo)
                                             | texto: Definir texto para "Player 2 WINS!"
                                             | Reiniciar: Definir visibilidade para Visível

19 - Sistema: VidaP2 = 0                     | Player2: Definir animação para "morte" (executar do início)
     Sistema: Ativa uma vez                  | Sistema: Aguardar 2 segundos (escala de tempo)
                                             | texto: Definir texto para "Player 1 WINS!"
                                             | Reiniciar: Definir visibilidade para Visível

20 - Reiniciar: Ao clicar                    | Sistema: Redefinir  variáveis globais para padrão
                                             | Sistema: Reiniciar o layout

👉 Quando a vida de um jogador chega a 0:

  • O personagem executa a animação de "morte".
  • O sistema aguarda 2 segundos.
  • O texto exibe a mensagem de vitória do oponente ("Player 1 WINS!" ou "Player 2 WINS!").
  • O botão Reiniciar fica visível.

👉 Ao clicar no botão Reiniciar, o sistema redefine todas as variáveis globais para os valores padrão e reinicia o layout, começando uma nova partida.

🏆 Assim, o jogo encerra quando a vida de um dos jogadores chega a 0, mostra o vencedor e permite reiniciar a luta.
